模块严格遵循 Clean Architecture 分层设计：
- **数据层 (Data)**: `user_repository.dart` 封装底层 API 调用，处理请求参数与响应解析。
- **领域层 (Domain)**: `user_use_cases.dart` 定义独立的业务用例（如获取信息、关注用户），隔离具体实现细节。
- **表现层 (Presentation)**: `user_controller.dart` 作为状态管理中心，协调用例执行并暴露响应式状态；`member_page.dart` 为入口页面，组合 `profile`、`seasons`、`coins`、`likes` 等原子化 Widget 进行 UI 渲染。
- **依赖方向**: 表现层依赖领域层，领域层依赖数据层，通过 `user.dart` 统一导出各层接口。
