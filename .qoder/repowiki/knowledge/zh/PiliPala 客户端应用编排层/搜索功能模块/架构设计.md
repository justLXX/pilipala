- 采用 Clean Architecture 分层设计，包含 data（数据层）、domain（领域层）和 presentation（表现层）三个子包。
- 数据层 `search_repository.dart` 封装网络请求，将 API 响应转换为领域模型。
- 领域层 `search_use_cases.dart` 定义业务用例（GetHotSearchUseCase, SearchContentUseCase, GetSearchSuggestionsUseCase），作为数据层与表现层的隔离边界。
- 表现层使用 GetX 状态管理，`search_controller.dart` 协调用例执行并暴露响应式状态，`search_page.dart` 及 widgets 子目录负责 UI 渲染。
- 模块入口 `search.dart` 作为 barrel file 统一导出各层公共接口。
