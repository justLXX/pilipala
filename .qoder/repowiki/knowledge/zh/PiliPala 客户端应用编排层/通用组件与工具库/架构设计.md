- **分层结构**：划分为 `common`（UI 层）和 `utils`（逻辑层）。`common/widgets` 封装了视频卡片、骨架屏等高频 UI 组件；`utils` 包含网络签名、存储管理、路由跳转等工具。
- **依赖方向**：工具类（如 `route_push.dart`）直接调用业务 HTTP 接口并耦合 `Get.toNamed` 进行导航，体现了工具层对业务逻辑的侵入。
- **关键实现**：通过 `constants.dart` 统一定义样式常量；`storage.dart` 基于 Hive 封装全局存储箱；`wbi_sign.dart` 实现了 Bilibili 特有的 Wbi 签名算法。
